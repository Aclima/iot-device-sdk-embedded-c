Google Cloud IoT Device SDK for Embedded C API

The Device SDK API securely connects client applications to Cloud IoT Core in order to publish and subscribe to messages via MQTT.

Getting started

To use the Device SDK API:

  1. Build the Device SDK on your device
  2. Create a client application with the Device SDK API methods

Device SDK API methods

You can use the Device SDK API to create a client application that connects to and communicates with MQTT brokers. The user guide demonstrates the typical client application workflow. It highlights the following Device SDK API methods:

  • Connecting to Cloud IoT Core
    • iotc_initialize()
    • iotc_create_context()
    • iotc_connect(iotc_context_handle_t iotc_h, const char *username, const char *password, const char *client_id, uint16_t connection_timeout, uint16_t keepalive_timeout, iotc_user_callback_t *client_callback)
  • Publishing and subscribing to messages
    • iotc_publish(iotc_context_handle_t iotc_h, const char *topic, const char *msg, const iotc_mqtt_qos_t qos, iotc_user_callback_t *callback, void *user_data)
    • iotc_publish_data(iotc_context_handle_t iotc_h, const char *topic, const uint8_t *data, size_t data_len, const iotc_mqtt_qos_t qos, iotc_user_callback_t *callback, void *user_data)
    • iotc_subscribe(iotc_context_handle_t iotc_h, const char *topic, const iotc_mqtt_qos_t qos, iotc_user_subscription_callback_t *callback, void *user_data)

See the methods index for a complete reference.

Dependencies

The Device SDK API uses the Device SDK Board Support Package (BSP). The Device SDK API depends on the hardware-specific drivers and routines in the BSP to implement the MQTT protocol.

The Device SDK has a BSP for POSIX platforms by default. To use the Device SDK API on non-POSIX platforms, you must first customize the BSP and port it to your device.
